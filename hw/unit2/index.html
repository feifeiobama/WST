  4. Every page mentioned above except Demo 2 passed HTML5 validation through the use of the W3C Markup Validator.

    Demo 2 failed because it contains the frameset element, which is obsolete in HTML5. The frameset elements were criticised in terms of usability and accessibility, some key points are as follows:

    • Users cannot properly reload, navigate, bookmark and share pages. Since the URL has generally not changed, users were often taken back to the default frameset.
    • Most search engines are unable to index frame-based websites properly. This often means that when someone finds your website, it might look wrong.
    • The use of too many frames can put a high workload on the server.

    The alternative solution to embed web pages, given by HTML standard, is either to use iframe and CSS instead, or to use server-side includes to generate complete pages with the various invariant parts merged in.
